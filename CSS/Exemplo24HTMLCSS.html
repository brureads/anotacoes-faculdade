Cores em CSS — Propriedades e Valores

Formatos de cor em CSS

1. Named colors — palavras-chave nomeadas

CSS oferece 148 nomes prontos como tomato, goldenrod, cornflowerblue. Fáceis de ler, mas sem controle fino de tom ou opacidade.

Propriedades usadas aqui: background-color: tomato · color: white · border-left: 5px solid goldenrod

2. Hexadecimal — #RRGGBB

O formato mais comum. Cada par representa um canal (Vermelho, Verde, Azul) com valores de 00 a FF. Com 8 dígitos (#RRGGBBAA), o último par define a opacidade.

Propriedades usadas aqui: background-color: #43054E · color: #fff

3. rgb() e rgba() — canais numéricos

Define cada canal (R, G, B) de 0 a 255. O 4º valor (alpha) controla a opacidade: 0 = transparente, 1 = opaco.

Propriedades usadas aqui: background-color: rgb(236, 0, 137) · color: rgba(255, 255, 255, 0.95)

4. hsl() e hsla() — matiz, saturação e luminosidade

Hue (0–360°) define a cor; Saturation e Lightness (0–100%) controlam a vivacidade e o brilho. Ideal para criar variações de uma mesma cor apenas ajustando o L.

Propriedades usadas aqui: background-color: hsl(210, 80%, 40%) · borda: hsl(210, 80%, 25%) (mesma cor, mais escura)

5. Variáveis CSS — Custom Properties

Defina a paleta uma vez em :root com prefixo -- e reutilize em qualquer lugar com var(--nome). Facilita manutenção e criação de temas.

Propriedades usadas aqui: background-color: var(--cor-secundaria) · border-left: 5px solid var(--cor-primaria)

6. Gradiente — linear-gradient()

Gradientes são valores da propriedade background (não background-color). Permitem transição suave entre duas ou mais cores, com direção ou ângulo definidos.

Propriedade usada aqui: background: linear-gradient(to right, #EC0089, #43054E)

Contraste e Acessibilidade (WCAG)

O padrão WCAG 2.1 exige razão mínima de 4,5:1 para texto normal (nível AA) e 7:1 para máxima acessibilidade (nível AAA). Ferramentas como o WebAIM Contrast Checker calculam essa razão automaticamente.

Bom contraste — nível AAA (~21:1)

Texto branco sobre fundo preto. Máxima legibilidade, amplamente usado em interfaces modernas.

Contraste mínimo aceitável — nível AA (~4,6:1)

Texto azul escuro sobre branco. Aprovado pelo WCAG, adequado para uso em produção.

Baixo contraste — reprovado (~1,1:1)

Amarelo claro sobre amarelo. Praticamente ilegível — prejudica todos os usuários, especialmente daltônicos.
